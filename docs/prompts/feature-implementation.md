# Feature Implementation Prompt

Read `AGENTS.md`, `ARCHITECTURE.md`, `docs/TOOLING.md`, `docs/FORMATTING.md`, the relevant product spec, and the active execution plan before editing.

## Instructions

1. Restate the acceptance criteria before making changes.
2. Implement the smallest coherent change that satisfies the task.
3. Update docs if behavior, constraints, or architecture changed.
4. Run `./scripts/preflight.sh`.
5. Review your own diff for bugs, regressions, missing tests, and architecture violations.
6. Fix what you find and rerun `./scripts/preflight.sh`.
7. Prepare a concise PR summary with risks and follow-ups.

## Output Shape

- Acceptance criteria
- Implementation summary
- Validation performed
- Risks
- Follow-ups
